## 프로젝트 흐름
1. 입력 질의를 받는다.
2. 질의를 분석한 후 전략 후보를 산출한다.
3. 사용자에게 전략을 선택하도록 한다. (전략 선택 UI 제공, 승인/거절/다른 전략 요청)
4. 선택한 전략으로 얻어낸 결과의 품질을 검증한다.
5. 문제가 있을 시 전략을 바꾸거나, 재검색하거나, 사용자에게 재질문한다.
6. 문제가 없다면 LLM에 근거 + 메타데이터를 전달해 답변을 생성한다. 

---
## 전략 설계
- SQL 템플릿
    - LOT / 장비ID / 날짜 / 수율 / 불량 Top / trend / 통계와 같은 정형 데이터 질문
- 벡터 RAG
    - ~란 뭐야, 기준, 절차, 규정, 설명
- 그래프 RAG 또는 하이브리드 RAG
    - 랑 B 관계, 원인-결과, 어떤 공정에서 영향을 받는 지 파악
- 벡터 우선 검색 
    - 특정 문서/업로드 파일 기반 질문

## 폴더별 용도
1. rest: 웹(HTTP) 레이어
2. agents: 오케스트레이션/전략선택/도구호출 등을 담당하는 레이어. 
3. schema: 테이블 정의
4. graph: GraphRAG. 그래프 구축/검색 알고리즘 등.
5. com: 공통. (설정/DB/레포/서비스/템플릿/유틸 등)
